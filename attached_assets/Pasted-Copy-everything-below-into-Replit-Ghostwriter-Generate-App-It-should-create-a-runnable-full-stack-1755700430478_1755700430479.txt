Database Schema (Prisma)

Create the following models with sensible indexes and relations:

User: id, name, username (unique), email (unique), passwordHash, whatsapp, address, location, bio (profile description), avatarUrl, role (enum: USER, ADMIN), createdAt, updatedAt.

Wallet: id, userId (FK), type (enum: PI, USDT_TRON, USDT_TON, USDT_BNB, USDT_SOL, USDT_AVAX), address, label, createdAt.

Listing: id, sellerId (FK User), type (enum: REAL_ESTATE, SHIPPING_SERVICE, PRODUCT, SERVICE), title, slug, description, priceCrypto (decimal), currency (enum: PI, USDT), network (enum: PI_MAINNET, TRON, TON, BNB, SOL, AVAX), images (string[]), location, isActive (bool), createdAt, updatedAt.

Escrow: id, listingId (FK), buyerId (FK), sellerId (FK), amount (decimal), currency (enum as above), network (enum as above), status (enum: CREATED, FUNDED, SHIPPED, DELIVERED, DISPUTED, RELEASED, REFUNDED), buyerTxHash (string? for funding), sellerTxHash (string? for payout), adminNote (string), platformFeePct (default 10), platformFeeAmount (decimal), sellerNetAmount (decimal), createdAt, updatedAt.

Follow: id, followerId (FK), followeeId (FK), status (enum: PENDING, ACCEPTED, REJECTED), createdAt.

Message: id, threadId, senderId (FK), recipientId (FK), content, readAt (DateTime?), createdAt. (Implement threads by deterministic threadId composed of two user ids sorted.)

Notification: id, userId (FK), type (enum: FOLLOW_REQUEST, MESSAGE, ESCROW_UPDATE, REVIEW), data (JSON), readAt?, createdAt.

BlogPost: id, authorId (FK), title, slug, excerpt, contentMarkdown, coverImageUrl, published (bool), createdAt, updatedAt.

PlatformWallet: id, type (enum: PI, USDT_TRON, USDT_TON, USDT_BNB, USDT_SOL, USDT_AVAX), address, createdAt, updatedAt. (Admin‑managed; used as escrow sink/source.)


Key Features & Flows

Marketplace

Public /marketplace with filters (type, price, currency/network, location, rating). Card grid with image, title, price, rating, seller.

/listing/[slug] detail page shows gallery, description, price, seller profile, average rating, reviews, Contact via WhatsApp button, Follow seller, DM button, and Buy via Escrow CTA.

Create/Edit listing (authenticated) at /sell/new and /sell/[id]/edit.


Escrow (10% platform fee)

When buyer clicks Buy via Escrow, create Escrow with status=CREATED and show instructions:

Display Platform wallet address for the selected currency/network (from PlatformWallet).

Buyer enters amount and later submits buyerTxHash once on‑chain payment is made.

Status moves to FUNDED after admin verifies (manual for MVP via admin dashboard action).

Seller ships/provides service; buyer marks DELIVERED.

Admin reviews and RELEASES escrow. Calculate platformFeeAmount = amount * 0.10 and sellerNetAmount = amount - platformFeeAmount. Store optional sellerTxHash if payout is on‑chain.

Support DISPUTED and REFUNDED manual paths in admin.


No automatic blockchain integration in MVP—record tx hashes only; admins verify manually.


Reviews

1–5 star with comment.

Only users with a RELEASED escrow for that listing or seller can review. Prevent duplicates per buyer/listing.


Social: Follow & DMs

DMs: authenticated users can message each other. Use Socket.IO if available; otherwise poll GET /api/messages?threadId=... every 5s. Show unread badges.
